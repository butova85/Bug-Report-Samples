# Bug Report Samples

Structured QA bug report examples for GIS/product-testing style workflows.

## Goal
Demonstrate clear, reproducible defect documentation using:
- concise titles
- reproducible steps
- expected vs actual results
- severity and impact rationale
- environment notes

## Status
**v1 in progress**  
Last updated: 2026-02-12

## Repository Contents
- `bug-reports.md` — sample bug reports
- `bug-template.md` — reusable report template
- `evidence/` — screenshots/log snippets (to be added)

## Severity Model
- Critical: system/data loss, major outage
- High: core function broken, no practical workaround
- Medium: significant issue with workaround
- Low: minor defect / cosmetic / low user impact
